# Chat Frontend Facts

**Maven module location**: The frontend Maven module descriptor lives at `java/chat-frontend/pom.xml`, but the npm project, build script, and generated `dist/` assets stay under `frontends/chat-frontend/`.

**LangChain event compatibility**: `useSseStream` and `chat-panel` normalize both canonical rich events (`eventType`) and raw LangChain envelopes (`event`, `data`) into the frontend `ChatEvent` model via `src/lib/langchain-events.ts`, so live streams and replayed `history/lc` entries render consistently.

**Required agent-app endpoint surface**: The UI currently uses these agent-app routes (directly or via generated client service wrappers): `POST /chat/{conversationId}`, `POST /v1/conversations/resume-check`, `GET /v1/conversations/{conversationId}/resume`, `POST /v1/conversations/{conversationId}/cancel`, conversation proxy routes (`GET/PATCH/DELETE /v1/conversations/{id}`, `GET /v1/conversations`, `GET /v1/conversations/{id}/entries`, `GET /v1/conversations/{id}/forks`, `POST /v1/conversations/search`), sharing routes (`GET/POST /v1/conversations/{id}/memberships`, `PATCH/DELETE /v1/conversations/{id}/memberships/{userId}`, `GET/POST/DELETE /v1/ownership-transfers`, `POST /v1/ownership-transfers/{transferId}/accept`), and attachment routes (`POST /v1/attachments` multipart only, `GET /v1/attachments/{id}/download-url`, `DELETE /v1/attachments/{id}`, `GET /v1/attachments/download/{token}/{filename}`), plus `GET /config.json`.
**Conversation sidebar ordering**: Preserve the order returned by `GET /v1/conversations`; do not apply a client-side sort in `src/App.tsx`, or the sidebar can drift from the server’s pagination order.
**Fork view semantics gotcha**: `src/lib/conversation.ts#createForkView` assumes `forkedAtEntryId` is the last inherited entry, not the first excluded entry. Fork markers are attached using the previous rendered entry ID, so an exclusive-stop backend change needs more than swapping the `break` condition in `getEntries(...)`.
