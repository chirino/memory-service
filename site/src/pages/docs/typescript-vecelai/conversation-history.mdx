**What changed**: The app imports `appendHistoryEntry(...)`, records the USER turn before model generation, and records the AI turn after generation.

**Why**: `memory` keeps model context; `history` keeps human-readable conversation turns for downstream UX and search features.

```bash
function get-token() {
  curl -sSfX POST http://localhost:8081/realms/memory-service/protocol/openid-connect/token \
    -H "Content-Type: application/x-www-form-urlencoded" \
    -d "client_id=memory-service-client" \
    -d "client_secret=change-me" \
    -d "grant_type=password" \
    -d "username=bob" \
    -d "password=bob" \
    | jq -r '.access_token'
}
```
