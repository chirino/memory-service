## Step 2: Enable Memory-Backed Conversations

**Starting checkpoint**: [typescript/examples/vecelai/doc-checkpoints/02-with-memory](https://github.com/chirino/memory-service/tree/main/typescript/examples/vecelai/doc-checkpoints/02-with-memory)

Checkpoint `02` is built from checkpoint `01` with three additions.

### Import the context memory helper

<CodeFromFile
  file="typescript/examples/vecelai/doc-checkpoints/02-with-memory/src/app.ts"
  lang="typescript"
  lines="4-4"
/>

**What changed**: The checkpoint imports `withContextMemory(...)` from `@chirino/memory-service-vercelai`.

**Why**: This helper gives you a ChatMemory-style workflow: read current memory, mutate in-process memory, and persist efficiently at the end.

The endpoint also forwards `Authorization: Bearer ...` to Memory Service calls so access control remains user-scoped.

### Change the endpoint to accept a `conversationId`

<CodeFromFile
  file="typescript/examples/vecelai/doc-checkpoints/02-with-memory/src/app.ts"
  lang="typescript"
  lines="25-27"
/>

**What changed**: The route changes from `POST /chat` to `POST /chat/{conversationId}`.

**Why**: The conversation ID in the URL becomes the storage key for memory entries. Calls to the same ID reuse context; a new ID starts fresh context.

### Use context memory in the chat flow

<CodeFromFile
  file="typescript/examples/vecelai/doc-checkpoints/02-with-memory/src/app.ts"
  lang="typescript"
  lines="37-60"
/>

**What changed**: The handler wraps generation in `withContextMemory(...)`, gets prior messages with `contextMemory.get()`, then calls `contextMemory.append(...)` for USER and AI messages.

**Why**: If you only append, the helper persists only the delta via append calls. If you call `contextMemory.clear()` or `contextMemory.set([...])`, it automatically switches to full sync (`/entries/sync`) to replace memory state.

You can also replace memory explicitly:

```typescript
contextMemory.clear();
contextMemory.set([
  { role: "USER", text: userMessage },
  { role: "AI", text: assistantText },
]);
```

Make sure Memory Service and Keycloak are running, then define a helper to get a user token:

```bash
function get-token() {
  curl -sSfX POST http://localhost:8081/realms/memory-service/protocol/openid-connect/token \
    -H "Content-Type: application/x-www-form-urlencoded" \
    -d "client_id=memory-service-client" \
    -d "client_secret=change-me" \
    -d "grant_type=password" \
    -d "username=bob" \
    -d "password=bob" \
    | jq -r '.access_token'
}
```
